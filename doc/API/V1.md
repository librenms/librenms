# API v1

API v1 is a modern REST API built with [Laravel Restify](https://restify.binarcode.com/). It follows the [JSON:API](https://jsonapi.org/) specification with built-in pagination, filtering, and search.

API v1 runs alongside the existing [v0 API](index.md). The v0 API continues to work as before.

## Authentication

API v1 uses [Laravel Sanctum](https://laravel.com/docs/sanctum) Bearer tokens.

### Managing tokens

Generate, list, and revoke tokens via the `lnms` CLI:

```bash
# Create a token for a user
lnms api:token create admin
lnms api:token create admin --token-name="monitoring-script"

# List all tokens for a user
lnms api:token list admin

# Revoke a token by ID (use list to find the ID)
lnms api:token revoke admin --token-id=3
```

Save the output of `create` — the plain-text token is only shown once.

### Using the token

Pass the token in the `Authorization` header:

```bash
curl -H 'Accept: application/vnd.api+json' \
     -H 'Authorization: Bearer YOUR_TOKEN_HERE' \
     https://librenms.example.com/api/v1/devices
```

For write requests (POST, PUT, PATCH), set the `Content-Type` header:

```bash
curl -X POST \
     -H 'Content-Type: application/vnd.api+json' \
     -H 'Authorization: Bearer YOUR_TOKEN_HERE' \
     -d '{"name": "new-group", "type": "static"}' \
     https://librenms.example.com/api/v1/device-groups
```

All responses use `Content-Type: application/vnd.api+json`. The server also accepts `application/json` as `Content-Type` for convenience. Requests with a body that use any other content type will receive `415 Unsupported Media Type`.

Unauthenticated or invalid requests return HTTP 401:

```json
{"message": "Unauthenticated."}
```

## Base URL

All v1 endpoints are under `/api/v1`.

## Response Format

Responses follow the [JSON:API](https://jsonapi.org/) structure with `Content-Type: application/vnd.api+json`.

### List (index)

```json
{
  "meta": {
    "current_page": 1,
    "from": 1,
    "last_page": 3,
    "path": "http://librenms.example.com/api/v1/devices",
    "per_page": 15,
    "to": 15,
    "total": 42
  },
  "links": {
    "first": "...?page=1",
    "next": "...?page=2",
    "path": "...",
    "prev": null,
    "filters": "/api/v1/devices/filters"
  },
  "data": [
    {
      "id": "1",
      "type": "devices",
      "attributes": {
        "hostname": "router01.example.com",

        "status": 1
      }
    }
  ]
}
```

### Single resource (show)

```json
{
  "data": {
    "id": "1",
    "type": "devices",
    "attributes": {
      "hostname": "router01.example.com",
      "os": "iosxe",
      "status": 1
    }
  }

```

## Endpoints

### Devices

| Method | Route | Description |
|--------|-------|-------------|
| GET | `/api/v1/devices` | List all devices |
| GET | `/api/v1/devices/{id}` | Get a single device |

Searchable fields: `hostname`, `sysName`, `os`

### Ports

| Method | Route | Description |
|--------|-------|-------------|
| GET | `/api/v1/ports` | List all ports |
| GET | `/api/v1/ports/{id}` | Get a single port |

Searchable fields: `ifName`, `ifAlias`, `ifDescr`

### Users

| Method | Route | Description |
|--------|-------|-------------|
| GET | `/api/v1/users` | List users (admin only) |
| GET | `/api/v1/users/{id}` | Get a single user |

Searchable fields: `username`, `realname`, `email`

### Alert Templates

| Method | Route | Description |
|--------|-------|-------------|
| GET | `/api/v1/alert-templates` | List all alert templates |
| GET | `/api/v1/alert-templates/{id}` | Get a single alert template |
| POST | `/api/v1/alert-templates` | Create an alert template (admin) |
| PUT/PATCH | `/api/v1/alert-templates/{id}` | Update an alert template (admin) |
| DELETE | `/api/v1/alert-templates/{id}` | Delete an alert template (admin) |

Searchable fields: `name`

### Alert Rules

| Method | Route | Description |
|--------|-------|-------------|
| GET | `/api/v1/alert-rules` | List all alert rules |
| GET | `/api/v1/alert-rules/{id}` | Get a single alert rule |
| POST | `/api/v1/alert-rules` | Create an alert rule (admin) |
| PUT/PATCH | `/api/v1/alert-rules/{id}` | Update an alert rule (admin) |
| DELETE | `/api/v1/alert-rules/{id}` | Delete an alert rule (admin) |

Searchable fields: `name`

Writable fields: `name`, `severity`, `disabled`, `proc`, `notes`, `invert_map`

Read-only fields: `rule`, `query`, `builder`, `extra`

### Device Groups

| Method | Route | Description |
|--------|-------|-------------|
| GET | `/api/v1/device-groups` | List all device groups |
| GET | `/api/v1/device-groups/{id}` | Get a single device group |
| POST | `/api/v1/device-groups` | Create a device group (admin) |
| PUT/PATCH | `/api/v1/device-groups/{id}` | Update a device group (admin) |
| DELETE | `/api/v1/device-groups/{id}` | Delete a device group (admin) |

Searchable fields: `name`, `desc`

Writable fields: `name`, `desc`, `type`

Read-only fields: `rules`

### Profile

| Method | Route | Description |
|--------|-------|-------------|
| GET | `/api/v1/profile` | Get the authenticated user's profile |

## Pagination

All list endpoints are paginated. Control pagination with query parameters:

```
GET /api/v1/devices?page=2&perPage=25
```

Default page size is 15.

## Search

Use the `search` query parameter to search across the resource's searchable fields:

```
GET /api/v1/devices?search=router
```

This searches across `hostname`, `sysName`, and `os` for devices.

## Access Control

- **Devices and Ports** — Users with global read access see all resources. Other users only see resources they have been granted access to.
- **Alert Templates** — Users with global read access can view. Only admins can create, update, or delete.
- **Alert Rules** — Users with global read access can view. Only admins can create, update, or delete. Results are scoped to devices the user has access to.
- **Device Groups** — Users with global read access can view. Only admins can create, update, or delete. Results are scoped to groups the user has access to.
- **Users** — Only admins can list all users. Non-admin users can only view their own profile.
